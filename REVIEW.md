## UserApiClient.java

### 1. Потокобезопасность синглтона

**Проблема:**
```java
public static UserApiClient getInstance(String baseUrl) {
    if (instance == null) {
        instance = new UserApiClient(baseUrl);
    }
    return instance;
}
```
# Что сломается: При параллельном запуске тестов два потока могут одновременно зайти в if (instance == null) 
#               и создать два разных экземпляра. Последующие вызовы будут работать с разными baseUrl.

# Риск: Тесты будут обращаться к разным окружениям, что приведет к нестабильным результатам.

# Предложение: Убрать синглтон, сделать конструктор публичным. Клиент не должен управлять жизненным циклом этого объекта.

### 2. deleteUser не возвращает Response
**Проблема:**
```
public void deleteUser(String userId) {
    given()...delete("/api/v1/users/" + userId);
}
```
# Что сломается: Нельзя проверить статус ответа. Тест не узнает, что удаление не удалось (404, 500).

# Предложение: Вернуть Response и использовать pathParam вместо конкатенации.


### 3. Неиспользуемое поле timeout

**Проблема:**
# Поле timeout = 5000 объявлено, но нигде не используется. 

# Предложение: Либо удалить, либо применить к RestAssured-запросам.(если она нужна для клииента, то стоит ее получать из кофнига, что бы можно было изменять ее)

### 4. Хардкод пути к драйверу

**Проблема:**
```
System.setProperty("webdriver.chrome.driver", "/usr/local/bin/chromedriver");
```
# Что сломается: Тесты не запустятся на Windows, на CI или на машине другого разработчика.

# Предложение: Использовать WebDriverManager или Selenium Jupiter (как сделано в исправленной версии).

### 5. Смешивание неявного и явного ожиданий

**Проблема:**
```
В BaseTest установлен implicitlyWait(10s), а в тестах добавляется WebDriverWait(driver, 15s).
```
# Что сломается:

# -Таймауты могут суммироваться (до 25 секунд)

# -Возникают StaleElementReferenceException при повторном поиске элемента

# -Тесты становятся flaky: элемент может появиться на 11-й секунде — иногда implicit его найдет, иногда explicit выбросит исключение

# Предложение: Удалить implicitlyWait полностью, использовать только явные ожидания WebDriverWait.

### 6. Аннотация @ExtendWith

**Проблема:**
```
Класс использует @ExtendWith(SeleniumExtension.class), но такого класса нет в проекте. Selenium Jupiter требует @ExtendWith(SeleniumJupiter.class).
```
# Предложение: Исправить импорт или удалить расширение, если не используется.
